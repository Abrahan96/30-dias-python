# 30 Días de Python 🐍

Repositorio personal de práctica progresiva para aprender Python desde los fundamentos hasta programación orientada a objetos y proyectos.

> El nombre “30 días” es referencial. El objetivo no es avanzar rápido, sino comprender cada tema antes de continuar.

## Objetivos

- Desarrollar lógica de programación.
- Comprender Python, no solo copiar código.
- Resolver ejercicios progresivos.
- Aprender a depurar errores.
- Aplicar buenas prácticas.
- Crear proyectos prácticos.
- Documentar el progreso en GitHub.

## Metodología

Cada tema seguirá este ciclo:

1. Concepto.
2. Explicación de la lógica.
3. Ejemplo guiado.
4. Ejercicio individual.
5. Revisión del código.
6. Mini-reto.
7. Reflexión y documentación.

## Ruta de aprendizaje

| Etapa | Contenido |
|---|---|
| 1 | Variables, tipos de datos, entrada y salida |
| 2 | Condicionales, bucles y validaciones |
| 3 | Listas, tuplas, diccionarios y conjuntos |
| 4 | Funciones, parámetros, retorno y módulos |
| 5 | Manejo de errores y archivos |
| 6 | Programación orientada a objetos |
| 7 | Proyectos integradores |

## Estructura de cada día

Cada carpeta puede contener:

- `README.md`: teoría, objetivo y reflexión.
- `ejercicio.py`: práctica principal.
- `reto.py`: mini-reto.
- `solucion.py`: solución final después de la revisión.
- `notas.md`: errores encontrados y aprendizajes.

## Regla principal

No avanzar porque el código funciona. Avanzar cuando se comprenda por qué funciona.
